**UVIJETI**
Što je uvijet?
- Uvijet je skup riječi koje govore što se treba ispuniti kako bi se izvšio niz/red naredbi
- Primjeri uvijeta:
	- AKO - if
	- DOK - while
	- Kada - when (ne postoji)
	- ZA - for
- Kako se zadovoljava uvijet? Sa provjerom znakova vrijednosti:
== jednako je - provjera jesu li vrijednosti jednake
= dodjeljuje vrijednost
!= različito je
|| (alt gr + W) ili - ili je jedna vrijednost točna ili je druga
&& i - obje vrijednosti moraju biti točne/istinite
> veće od
< manje od
<= manje ili jednako od
>= veće ili jednako od

primjeri if petlji:
if(1 +1 == 2) --> istina
if(2 + 3 != 4) --> istina
if(18 > 19) --> laž
if(21 <= 21) --> istina

kao da piše:
istina -> 1 + 1 = 2
laž -> 5 + 5 = 11
if(istina || laž) --> istina
if(istina || istina) --> istina
if(laž || istina) --> istina
if(laž || laž) --> laž
if(istina && istina) --> istina
if(istina && laž) --> laž

*METODE*
- metoda je skup narebi koje se trebaju odraditi u određenim uvjetima i intervalu

Ključne metode u Unityu:

- Awake 
	- odvija se prije 0 framea
	- događa se prija stvaranje bilokakvih objekata na sceni
	- izvršava se samo jedan put

- Start
	- izvršava se netom prije prvog framea
	- izvršava se samo jedan put

- Update
	- svaki frame se jednom izvrši

kako se piše metoda:

void nazivMetode()
{
	linija naredbe;
}

* vidljivost metoda/varijabli
- public - ispred vrste varijable kojoj omogucavamo da mi kao developeri možemo unositi i mjenjati vrijednost varijable u Unity Inspectoru ili pozivati iz drugih skripti
npr. public float primjer = 4.7f;

- private - stavlje se ispred varijable i onemogucava vidjljivost i promjene vrijednosti varijable iz Unity Inspectora i pozivanje iz drugi skripti. Ako ispred varijable/metode ne piše ništa, automatski je private.

- ("") - unutar navodnika tekst za ispis, poslje može varijabla za ispis (spajanje se radi sa pomoću znaka +)

*NOVA PETLJA - For petlja

for => dok je uvijet zadovoljen obavlja naredbe
int i => brojac, nemora se zvati i, ali se tako najcesce zove
uvijet => uvijet koji mora biti zadovoljen
inkrement => utjece na brojac, nareba s kojom djelujemo na brojac na više
dekrement => utjece na brojac, nareba s kojom djelujemo na brojac na manje

nacin pisanja for petlje:

for(brojac; uvijet; inkrement)
{
	naredba;
}

primjer:
